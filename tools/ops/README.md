# Операционные проверки

Одноразовые скрипты для ручной диагностики. Запускаются из корня репозитория
через `npx tsx`, переменные окружения — как у приложения (`.env`).

## fd-sync-check.ts — прогон синка football-data

Выполняет один полный проход `syncFootballData` против БД из `DATABASE_URL`
и печатает исход (fixtures/results/live/unmatched) + состояние матча №1.
Идемпотентен: второй запуск подряд должен дать нули.

```bash
FD_TOKEN=... npx tsx tools/ops/fd-sync-check.ts        # локальная БД из .env
```

## sheet-verify.ts — чтение выгрузки Google Sheets

Читает первые строки каждой вкладки экспортированной таблицы, чтобы глазами
проверить формат (русские заголовки, флаги, сетка победителей групп).

```bash
GOOGLE_SA_JSON='{"...":"..."}' SHEET_ID=... npx tsx tools/ops/sheet-verify.ts
```

На проде креды лежат только в `/opt/toto/.env` — удобнее выполнять проверку
изнутри контейнера, не вынося ключ с сервера.
